# Правила сжатия транскрипта в конспект (kb/tokovinin_kb.md)

Это смысловой шаг — компрессия делается моделью, читающей `<id>_full.txt`
(вывод `clean_transcript.py`) целиком, а не скриптом. Ориентировочное сжатие:
**~6x по токенам** (проверено на a8a-hy31pdI: ~10 900 → ~1 760 токенов, cl100k_base).

## Что выбрасывать полностью
- Рекламный блок — **не вырезается никаким скриптом** (нет визуальной границы,
  чтобы резать по тексту надёжно, см. Design notes в `SKILL.md`). Ориентир —
  флаг `possible_sponsor_intro` и его `approx_end_sec` из `<id>.flags.json`:
  если флаг true, пропустите рекламный контент в начале транскрипта по смыслу
  (обычно легко узнаётся при чтении), не полагаясь на точный таймкод.
- Звуковые/эмоциональные теги: `[музыка]`, `[смех]`, `[фыркает]`, `[тяжело вздыхает]`.
- Слова-паразиты, самоперебивы, повторы ("ну", "короче", "блин", "э-э", "как бы").
- Дословный пересказ реплик из чужого ролика — оставлять только **суть тезиса**,
  на который реагирует Токовинин, не диалог целиком.

## Что оставлять
- Позиция/аргумент самого Токовинина по каждой теме.
- Конкретные цифры и факты, если он их называет (сохранять как есть, не перефразировать).
- Редкие ёмкие цитаты/афоризмы (не больше 1-2 на главу).
- Явное разделение: тезис источника (`ролик-источник: ...`) vs. позиция автора
  (`Токовинин: ...`) — это главная ценность конспекта.

## Структура блока для одного видео

```markdown
## Название видео
id: <video_id> | <длительность> | https://www.youtube.com/watch?v=<video_id>
Тема: одна строка о формате видео (реакция/монолог/интервью)
Реклама: <бренд>, <диапазон таймкодов> (просто отметка, не пересказ)

### <Глава 1 — из chapters info.json, с таймкодом>
Ролик-источник: <1-2 предложения о тезисе/провокации>
Токовинин:
- тезис
- тезис
- цифра/факт, если был

### <Глава 2 — ...>
...
```

## Правила накопления в kb/tokovinin_kb.md
- Каждое новое видео — новый `## Название` в конец файла. Старые блоки не редактировать
  (кроме явных исправлений по запросу пользователя).
- Не создавать отдельный файл на видео — весь кб живёт в одном файле, чтобы
  модель могла получить контекст по всем видео разом без лишних round-trip'ов.
- Если видео короткое/не реакция (трейлер, монолог) — шаг с host/clip разметкой
  не нужен, весь текст = слова автора.
