# Флоу обработки видео Токовинина (черновик)

## 0. Вход
Ссылка на видео вида `https://www.youtube.com/watch?v=<ID>`.

## 1. Скачать субтитры (транскрипцию)
```
yt-dlp --skip-download --write-auto-sub --write-sub --sub-lang "ru,en" \
  --sub-format vtt -o "transcripts/<ID>" "<url>"
```
Взять `<ID>.ru.vtt` как основной источник (авто-субтитры на русском).
Дополнительно вытащить метаданные:
```
yt-dlp --skip-download --write-description --write-info-json -o "transcripts/<ID>_meta" "<url>"
```
Из info.json нужны: `title`, `duration`, `chapters` (таймкоды глав из описания), `channel`.

## 2. Очистить VTT в связный текст
- Убрать таймкоды, номера блоков, служебные строки (`WEBVTT`, `Kind:`, `Language:`).
- Убрать теги вида `<00:00:00.000><c>...</c>` regexp'ом `<[^>]+>`.
- html.unescape (там встречается `&gt;&gt;` — это маркер смены говорящего в авто-субтитрах, заменять на явный разделитель, напр. ` — `).
- Схлопнуть дубли соседних строк (авто-субтитры дают "бегущий" повтор слов).

## 3. Найти и вырезать рекламный блок
Наблюдение (на примере a8a-hy31pdI): реклама у Токовинина всегда в начале ролика (спонсор — Точка Банк), заканчивается фирменным экраном-дисклеймером на **фиолетовом фоне** с текстом "Реклама. Рекламодатель ...".
Практический способ найти границу:
1. Скачать видео в низком разрешении (`-f "bv*[height<=480]+ba/b[height<=480]"`).
2. Нарезать кадры через ffmpeg (`fps=1/8` для грубого прохода, `fps=2` для точной границы).
3. Детектировать фиолетовый экран программно: пиксель в центре кадра ≈ `(122, 51, 219)`.
4. Диапазон с фиолетовым экраном + куски до/после (ресторан/лого до, затемнение после) = граница рекламы. В примере: 0:00–~0:20.
5. Вырезать из текста реплики, попадающие в этот диапазон по таймкодам VTT.
Не проверено: всегда ли реклама только в начале, или бывает мид-ролл — при обработке нового видео стоит явно прогнать поиск фиолетового экрана по всему ролику, а не только в начале.

## 4. Разметить "сам автор" vs "вставка чужого ролика"
Формат канала — реакция: чередуются полноэкранные куски (только Токовинин) и split-screen (справа — вставка ролика с вшитыми субтитрами).
Способ:
1. Кадры каждые ~8 сек (шаг можно уменьшить для точности границ).
2. Проверить пиксель в правом верхнем углу кадра (`x=0.98*w, y=0.05*h`): если тёмный/однородный (RGB ~20,20,20, разброс каналов <8) → split-screen (вставка). Если светлый/цветной (книжная полка, фон кабинета) → полный экран (сам автор).
3. Схлопнуть в диапазоны (start, end, label), сопоставить с таймкодами VTT-реплик → каждой реплике присвоить метку `ТОКОВИНИН (сам)` / `ВСТАВКА (чужой ролик)`.
Оговорка: во время split-screen автор иногда комментирует поверх замороженного кадра — метка в эти секунды может быть неточной (шаг разметки даёт погрешность в несколько секунд).

## 5. Сжать в конспект для kb/tokovinin_kb.md
Это смысловой, не механический шаг — делается "вручную" (LLM читает labeled-транскрипт целиком и пишет сжатую версию), по главам из `chapters` (info.json) или по видимым тематическим блокам.
Правила сжатия (см. шаблон в самом kb/tokovinin_kb.md):
- Реклама — не включать вообще.
- Чужой ролик — не пересказывать дословно, только суть тезиса/провокации, на которую реагирует автор.
- Слова автора — тезисы, аргументы, цифры/факты, редкие ёмкие цитаты. Без воды, шуток, слов-паразитов, повторов.
- Каждое видео — один `## Название` блок, дописывается в конец файла, старые блоки не трогать.
- В конце блока — обязательно `id`, длительность, ссылка (чтобы можно было вернуться к первоисточнику).

## 6. Файлы проекта
```
Tokovinin/
├── channel_videos.txt              # список видео канала (yt-dlp --flat-playlist)
├── transcripts/
│   ├── <ID>_full.txt                # очищенная полная транскрипция
│   └── <ID>_labeled.txt             # с разметкой автор/вставка
└── kb/
    └── tokovinin_kb.md              # накопительный сжатый конспект по всем видео
```
